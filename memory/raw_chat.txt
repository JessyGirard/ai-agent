GET requests retrieve data and should return 200 on success.

POST requests create new resources and typically return 201 on success.

PUT requests replace a resource and should return 200 or 204 on success.

PATCH requests partially update a resource and should return 200 or 204 on success.

DELETE requests remove a resource and should return 200 or 204 on success.

A 400 status code means the request is malformed or invalid.

A 401 status code means authentication is required or missing.

A 403 status code means access is forbidden even with authentication.

A 404 status code means the requested resource was not found.

A 405 status code means the HTTP method is not allowed for the endpoint, and the Allow header specifies valid methods.

A 415 status code means the request has an unsupported media type, often due to incorrect Content-Type.

The Content-Type header specifies the format of the request body, such as application/json.

The Authorization header is used to send credentials such as API keys or bearer tokens.

Negative testing involves sending invalid or unexpected inputs to verify that the API correctly handles errors.

Positive testing verifies that valid requests return expected successful responses.

Boundary testing checks limits such as maximum field length or numeric ranges.

Rate limiting can cause a 429 status code when too many requests are sent in a short time.
